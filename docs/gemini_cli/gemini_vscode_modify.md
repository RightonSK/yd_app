# Gemini CLI: ファイルの作成・修正依頼方法 (VS Code環境)

このドキュメントでは、VS Code 環境で Gemini CLI を使ってファイルを作成・修正する際の具体的な命令方法を説明します。特に、相対パスを使ってサブディレクトリ内のファイルを指定する方法に焦点を当てます。

## 1. ファイルの作成依頼方法

新しいファイルを特定の場所に作成するよう依頼する場合、`file_path` を明確に指定することが重要です。

### 基本的な書式

`[ディレクトリパス]/[ファイル名]` の形式で、作成したいファイルのパスとファイル名を指定します。

### 例

*   **現在の作業ディレクトリ直下のサブディレクトリにファイルを作成する場合:**
    （例: `gemini_cli/` ディレクトリが現在の作業ディレクトリの場合）
    ```
    `./my_new_folder/my_script.py` というファイルを `./my_new_folder/` ディレクトリに作成し、簡単なPythonスクリプトを記述してください。
    ```
    **注意:** `my_new_folder` が存在しない場合、Gemini CLI は `mkdir` コマンドの使用を提案するか、あるいはファイル作成に失敗します。事前にディレクトリを作成しておくか、Gemini CLI にディレクトリ作成の指示も合わせて出しましょう。

    ```
    `./my_new_folder/` ディレクトリを作成し、その中に `my_script.py` というPythonスクリプトファイルを作成してください。内容は「print("Hello from Gemini CLI")」です。
    ```

*   **相対パスを使って、一つ上の階層のディレクトリにファイルを作成する場合:**
    （例: 現在の作業ディレクトリが `docs/gemini_cli/` の場合）
    ```
    `../images/example.png` というファイルを作成し、プレースホルダー画像を挿入してください。（実際には画像生成ツールが必要です）
    ```

*   **特定のパスにファイルを一から作成し、内容も指定する場合:**
    ```
    `lib/data/user_repository.dart` というファイルを作成し、`UserRepository` クラスと `fetchUser` メソッドのひな形をDartで記述してください。
    ```

## 2. ファイルの修正依頼方法

既存のファイルを修正するよう依頼する場合も、対象ファイルを明確に指定し、修正内容を具体的に記述することが重要です。

### 基本的な書式

`[対象ファイルのパス]` を指定し、その後で `[修正内容の詳細]` を続けます。

### 例

*   **現在の作業ディレクトリ直下のファイルを修正する場合:**
    （例: `gemini_cli/` ディレクトリが現在の作業ディレクトリの場合）
    ```
    `./settings.json` ファイルの `theme` プロパティの値を `"dark"` から `"light"` に変更してください。
    ```

*   **相対パスを使って、一つ下のディレクトリにあるファイルを修正する場合:**
    （例: 現在の作業ディレクトリが `docs/` の場合）
    ```
    `./gemini_cli/gemini_vscode_guide.md` ファイルの「基本的な使い方」セクションに、Gemini CLI の起動コマンド `gemini` を追記してください。
    ```

*   **相対パスを使って、一つ上の階層のディレクトリにあるファイルを修正する場合:**
    （例: 現在の作業ディレクトリが `docs/gemini_cli/` の場合）
    ```
    `../requirements_definition.md` ファイルの「3.1. 開発言語・フレームワーク」セクションに、「テストフレームワーク: flutter_test」という項目を追加してください。
    ```

*   **現在開いているファイルを修正する場合:**
    ```
    現在開いているファイル内の `calculateTotal` 関数に、消費税計算のロジックを追加してください。税率は8%とします。
    ```

*   **特定のコードブロックの修正を依頼する場合:**
    ```
    `lib/widgets/user_card.dart` ファイル内の `_buildAvatar` メソッドで、アバターのサイズを `40.0` から `50.0` に変更してください。
    ```

## 3. ヒント

*   **具体的なコードの提供**: 修正内容が複雑な場合は、修正後のコード例をプロンプト内に含めることで、Gemini CLI がより正確に意図を理解しやすくなります。
*   **文脈の提供**: なぜその変更が必要なのか、変更によって何が解決されるのかなど、変更の背景を伝えることで、より適切な修正が期待できます。
*   **ツールの承認**: Gemini CLI が提案する `replace` や `write_file` などのツール実行時には、必ず内容を確認し、問題がなければ承認してください。
