# VS CodeでGemini CLIを利用する手順

## 1. はじめに

このドキュメントは、Visual Studio Code (VS Code) の統合ターミナルを通じて Gemini CLI を効果的に利用するための手順を説明します。Gemini CLI と VS Code の連携により、開発者はよりスムーズにコードの生成、修正、分析を行うことができます。

## 2. 前提条件

*   **Gemini CLI のインストール**: お使いのシステムに Gemini CLI がインストールされていることを確認してください。
    *   インストール方法については、Gemini CLI の公式ドキュメントを参照してください。
*   **Visual Studio Code のインストール**: VS Code がインストールされており、利用できる状態であることを確認してください。

## 3. 基本的な使い方

VS Code の統合ターミナルから Gemini CLI を起動し、命令を与えます。

1.  **VS Code でプロジェクトを開く**: 関連するファイルやディレクトリが VS Code のエクスプローラーに表示されている状態にします。
2.  **統合ターミナルを開く**: VS Code のメニューから `ターミナル` > `新しいターミナル` を選択するか、`Ctrl+Shift+@` (Windows/Linux) または `Cmd+Shift+@` (macOS) で開きます。
3.  **Gemini CLI を起動する**: ターミナルで `gemini` または設定されたエイリアスを入力し、Enter を押します。

    ```bash
    gemini
    ```
4.  **命令を入力する**: Gemini CLI が起動したら、プロンプトが表示されるので、実行したいタスクを自然言語で入力します。
    例: `このプロジェクトの構成を調べて、概要を教えてください。`

5.  **エージェントの応答とツールの実行**:
    *   Gemini CLI はあなたの命令を解析し、適切なツール（ファイルの読み込み、シェルコマンドの実行など）を提案します。
    *   ツールを実行する前に、Gemini CLI はそのツールの内容と影響を確認するメッセージを表示します。内容を確認し、問題がなければ `y` (Yes) を入力して実行を許可します。

## 4. VS Code との連携 (エディターコンテキスト)

Gemini CLI は、VS Code のアクティブなエディターのコンテキスト（現在開いているファイル、カーソル位置など）を自動的に取得し、これを考慮して応答や作業を行います。

*   **アクティブファイル**: あなたが現在 VS Code で開いているファイルは、Gemini CLI の命令コンテキストとして利用されます。例えば、「このファイルを修正して」という命令では、現在アクティブなファイルが対象となります。
*   **カーソル位置**: 特定のコードブロックや行にカーソルがある場合、その情報もコンテキストとして提供され、より精度の高い指示が可能です。
    例: 特定のファイルの特定の行にある関数について質問する場合、「現在開いているファイルのカーソル位置にある関数を説明してください」のように指示できます。

## 5. 具体的な命令の仕方

より効果的に Gemini CLI を活用するための命令例です。

*   **コードの修正依頼**:
    `このファイルのバグを修正してください。エラーメッセージは「TypeError: Cannot read property 'x' of undefined」です。`
    `現在選択しているコードブロックのリファクタリングを提案し、適用してください。`

*   **ファイルの作成依頼**:
    `lib/utils/ 配下に `helper.dart` という名前で新しいユーティリティファイルを作成し、`formatDate` 関数を実装してください。`
    `docs/gemini_cli/ ディレクトリに、Gemini CLIのVS Codeガイドを記述したMarkdownファイルを作成してください。`

*   **コードの説明依頼**:
    `現在開いているファイルの `_buildWidget` 関数について詳しく説明してください。`
    `このプロジェクトの `pubspec.yaml` ファイルの依存関係について教えてください。`

*   **既存ファイルの変更依頼**:
    `lib/main.dart の `MyApp` クラス内で、テーマカラーを青から緑に変更してください。`
    `src/components/button.js ファイルの `onClick` ハンドラに、ユーザーの同意確認ロジックを追加してください。`

*   **ファイル内容の検索**:
    `このプロジェクト全体で `TODO` コメントがどこにあるか検索してください。`
    `lib ディレクトリ内の `.dart` ファイルで `setState` が使われている箇所を全てリストアップしてください。`

## 6. ヒントとベストプラクティス

*   **具体的で明確な指示**: あいまいな指示よりも、具体的で明確な指示を与える方が、Gemini CLI は意図したとおりの作業を実行しやすくなります。
    （例: 「ファイルを修正して」よりも「`lib/models/user.dart` の `User` クラスに `email` プロパティを追加してください。」）
*   **ファイルパスの指定**: 可能であれば、対象となるファイルのパスを正確に指定してください。相対パス（例: `lib/src/my_file.dart`）または絶対パスで指定できます。
*   **ツールの使用許諾プロンプト**: Gemini CLI がツール（`write_file`, `run_shell_command` など）を実行しようとする際には、必ずあなたの確認を求めます。内容をよく確認し、安全であることを確認してから `y` を入力してください。
*   **作業完了後の確認**: Gemini CLI が作業を完了したら、意図したとおりに変更が適用されているか、手動で確認することをお勧めします。

このガイドが、VS Code での Gemini CLI の利用に役立つことを願っています。
